입력 양식들

서버로 데이터를 전송하기 위해 사용
form 태그를 사용

form 태그의 속성

  • action : 이동 경로 url
  • method
    데이터를 전송 방식을 지정한다.
    get(기본) 또는 post
    생략시 get
  • name : 폼의 이름
  • id : 폼의 아이디

전송 방식

form 태그의 method 속성을 이용하여 지정
생략하면 get 방식으로 전송됨

GET

URL 주소 뒤에 파라미터를 붙여서 데이터 전달
길이 제한/북마크/뒤로가기 O 보안 X

POST

HTTPRequest 헤더에 포함시켜서 전달
길이제한/북마크/뒤로가기X 보안 O
  • input
    • text : 문자열을 표시할수있는 텍스트 박스
    • password: 패스워드 입력할수 있는 박스
    • submit : 입력된 값을 제출하는 버튼
    • button : 사용자 지정의 버튼 생성
    • checkbox : 사용자가 체크할수있는 박스, 이름을 하나로 지정하면 같은 이름으로 넘어감
      checked 입력해두면 체크가 된상태로 나타난다.
    • radio : 이름이 같아야 중복 체크가 안됨,체크박스와 같은 동그라미 버튼
    • file : 나중에 업로드를 구현해야됨
    • hidden: 사용자는 보이지 않지만 전송되어야되는 값은 히든속성을 사용하여 전송함
    • reset: 다시쓰기
    • image: 이미지는 이미지를 넣어 submit과 같은기능을 함
    • name : 서버에 데이터를 보낼때 파라미터의 이름으로 넘어간다. **꼭 써야됨
  • textarea: 텍스트 입력 박스를 만든다.
  • select: 셀렉트를 만들 수 있는 창을 만든다. optgroup을 하면 그룹으로 라벨화하여 여러 선택지를 나눌수 있다.
  • button: submit과 같은기능을 기본으로한다.
    type속성에 button이라고 속성지정하면 제출하지 않는다.
  • label: 글씨를 클릭해도 선택지가 선택됨, 선택지의 id를 라벨의 for와 연결한다
  • fieldset: 박스를 만들어서 구분함
  • lenged : 박스의 이름을 만듬
